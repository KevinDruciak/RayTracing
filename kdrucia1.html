<h2 id="notes-on-implementation-">Notes on implementation:</h2>

<h4 id="new-command-line-parameters-">New Command Line Parameters:</h4>

<ul>

<li><h5 id="-jitter">--jitter</h5>

<ul>

<li>the number of samples to average in jitter supersampling</li>

</ul>

</li>

<li><h5 id="-sr">--sr</h5>

<ul>

<li>the radius of the jittering samples</li>

</ul>

</li>

<li><h5 id="-radius">--radius</h5>

<ul>

<li>the radius of the light sampling for soft shadows</li>

</ul>

</li>

</ul>

<h4 id="known-bugs-">Known Bugs:</h4>

<ul>

<li>Directional light does not draw shadows if any element in the position vector is 0. It works if we add some epsilon to those zeros.</li>

<li>There is noticable artifacting in the larger models. I assume this is because of floating point errors.</li>

<li>Bounding box intersection collides with whole box, not just front faces and is thus slower. This also means that I have about twice as many bounding box intersections than ideal.</li>

<li>I cast more rays than in the examples. I assume this has to do poorly-defined termination criteria for recursion.</li>

<li>Dog looks atrocious.</li>

</ul>
